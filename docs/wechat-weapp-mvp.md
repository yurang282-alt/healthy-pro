# Healthy Pro 微信小程序端 MVP

## 目录

小程序端目录为：

`healthy-pro-weapp`

不用 `miniprogram` 作为目录名，避免和其他项目混在一起不好找。

## 当前策略

- PWA 继续保留，仍然是现有线上版本。
- 小程序端单独开发，第一阶段先用 mock 数据。
- 后端优先使用微信云开发，等拿到云开发环境 ID 后接入。
- 小程序端第一版数据暂不和 PWA/Supabase 互通。

## 第一阶段范围

- 基础评估
- 生成训练计划
- 根据训练记录重新调整计划
- 编辑计划
- 查看上一版和恢复旧版
- 今日训练首页
- 训练记录
- 器械库
- 我的页面

## 第二阶段范围

- 微信登录
- 云数据库
- 反馈
- 好友和排行
- 体验版真机测试

## 用户需要准备

- AppID：当前 mock 预览可用 `touristappid`；微信登录、云数据库和体验版上传需要正式小程序 AppID。
- 云开发环境 ID：待提供
- 微信开发者工具：已安装后导入 `healthy-pro-weapp`

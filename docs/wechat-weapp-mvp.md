# Healthy Pro 微信小程序端 MVP

## 目录

小程序端目录为：

`healthy-pro-weapp`

不用 `miniprogram` 作为目录名，避免和其他项目混在一起不好找。

## 当前策略

- PWA 继续保留，仍然是现有线上版本。
- 小程序端单独开发，当前已从 mock 版推进到 CloudBase/openid 同步雏形。
- 本地和云端数据都应使用 `docs/data-contracts.md` 的跨端字段，避免后续迁移临时字段。
- 后端优先使用微信云开发；当前代码已配置 AppID、CloudBase 环境、登录云函数和基础云端读写，后续重点是权限规则、真机验证和体验版发布。
- 小程序端第一版数据暂不和 PWA/Supabase 互通。

## 第一阶段范围

- 基础评估
- 生成训练计划
- 根据训练记录重新调整计划
- 编辑计划
- 查看上一版和恢复旧版
- 今日训练首页
- 训练记录
- 器械库
- 我的页面

## 已进入第二阶段

- 微信登录
- 云数据库
- 反馈
- 好友和排行

## 下一步验证范围

- CloudBase 集合权限规则
- 两个微信用户的数据隔离
- 换设备、清缓存、重装后的恢复
- 体验版真机测试和上传
- 小程序内更新公告可见性

## 用户需要准备

- AppID：当前代码已配置正式小程序 AppID；微信登录、云数据库和体验版上传必须使用正式 AppID。
- 云开发环境 ID：当前代码已配置 CloudBase 环境；发布前要核验该环境的集合权限和真实数据。
- 微信开发者工具：已安装后导入 `healthy-pro-weapp`
